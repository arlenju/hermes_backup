# 中文 AI 味特征库

本文件列出中文 LLM 生成文本中高频出现的"AI 味"特征，供润色时逐项排查。
持续补充——每次发现新的 AI 味模式都应追加到此文件。

## 一、结构模板化

| 特征 | 示例 | 改法 |
|------|------|------|
| 三段论 | "首先…其次…最后…" | 用自然过渡词，或直接删掉序号 |
| 总分总 | 开头一句总起→展开→"综上所述" | 删掉首尾的总起/总结句，让内容自己说话 |
| 对称排比 | "不仅是…更是…" / "既…又…还…" | 打破对称，只留最有信息量的一句 |
| 每段都是"主题句+展开+小结" | 段尾"由此可见…" | 删小结句，让段尾留白 |

## 二、填充短语（最明显的 AI 味）

| AI 味短语 | 为什么不好 | 替代方案 |
|-----------|-----------|---------|
| 值得注意的是 | 空转，无信息 | 直接说那个值得注意的事实 |
| 需要指出的是 | 同上 | 删掉，直接陈述 |
| 众所周知 | 套话 | 删掉 |
| 毋庸置疑 | 套话 | 删掉 |
| 总而言之 | 正式但空洞 | 用"所以"或直接给结论 |
| 综上所述 | 正式但空洞 | 同上 |
| 在…方面 | 翻译腔 | 直接说"在X上"或去掉 |
| 通过…的方式 | 翻译腔 | "用X"或直接说动词 |
| 从…的角度来看 | 翻译腔 | "从X看"或删掉 |
| 对…产生了深远的影响 | 空泛 | 给出具体影响是什么 |
| 发挥着重要作用 | 空泛 | 说清楚什么作用 |
| 在当今社会 | 套话开头 | 删掉，直接进主题 |
| 随着科技的不断发展 | 套话开头 | 删掉 |

## 三、递进煽情

| 特征 | 示例 | 改法 |
|------|------|------|
| 递进煽情 | "这不仅仅是X，更是Y" | 保留信息量大的那个，删煽情 |
| 假升华 | "这标志着…的新时代" | 删掉，或换成具体事实 |
| 过度修饰 | "非常""极其""十分""尤其" 堆砌 | 去掉副词，让名词/动词自己有力度 |

## 四、过度解释

| 特征 | 改法 |
|------|------|
| 每个术语后都跟一句解释 | 只在读者真正可能不懂时解释 |
| 每个观点后都举例 | 挑最有力的一两个例，不要全列 |
| 反复重申同一观点 | 说一次就够 |
| "换句话说" 反复出现 | 最多用一次，通常不需要 |

## 五、语气不匹配

| 场景 | AI 默认语气 | 应该用 |
|------|-----------|--------|
| 微信消息 | 书面、完整句 | 短句、口语、可省主语 |
| 汇报 | 平铺直叙 | 结论先行、数据支撑 |
| 邮件 | 过度礼貌（"恳请""烦请"堆叠） | 礼貌但不卑微，重点突出 |
| 公众号 | 学术化 | 有节奏、有金句、段落短 |
| 技术文档 | 口语化或过度正式 | 简洁专业、术语精准 |

## 六、逻辑连接词滥用

AI 倾向于每段开头都加连接词："此外""另外""然而""因此""与此同时"。

改法：
- 连续两段以上都用"此外/另外"时，去掉后面那个
- "然而"只在真正转折时用，不要用"然而"做段落开头装饰
- "因此"只在确实有因果时用

## 七、词汇偏好

AI 高频使用但真人少用的词（中文）：

| AI 高频 | 真人更常用 |
|---------|-----------|
| 赋能 | 帮助 / 支持 |
| 闭环 | 搞定 / 完成 |
| 抓手 | 方式 / 方法 |
| 颗粒度 | 细节 / 精度 |
| 对齐 | 确认一致 |
| 沉淀 | 积累 / 留下 |
| 打通 | 连通 |
| 兜底 | 保底 |
| 拉通 | 协调 |
| 矩阵 | 组合 / 体系 |

注意：如果用户原文就在用这些词（互联网行业常见），不要强行替换——
润色不是改词癖，是让表达更自然。

## 八、检查清单

润色完成前，逐项检查：

- [ ] 没有"首先其次最后"三段论
- [ ] 没有"值得注意的是""需要指出的是"等填充短语
- [ ] 没有递进煽情（"不仅仅是…更是…"）
- [ ] 副词没有堆砌（"非常""极其"不超过1次/段）
- [ ] 连接词没有段段开头都用
- [ ] 术语没有过度解释
- [ ] 语气匹配目标风格
- [ ] 没有改变原文事实信息

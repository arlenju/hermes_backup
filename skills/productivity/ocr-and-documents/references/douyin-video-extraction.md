# Douyin (抖音) Video Content Extraction

## When to use

User shares a `https://v.douyin.com/XXXXX/` short link and asks to research/analyze/summarize the video content.

## Technique: Browser + Console JavaScript

Douyin's web player doesn't expose video metadata in the accessibility tree reliably. The proven method is:

1. **Navigate** to the short URL with `browser_navigate` — it redirects to `https://www.douyin.com/video/<video_id>`
2. **Scroll down** once with `browser_scroll` to trigger lazy-loading of video metadata
3. **Extract content via `browser_console`** with JavaScript expressions:

```javascript
// Get page title and meta
JSON.stringify({
  title: document.title,
  meta: document.querySelector('meta[name="description"]')?.content,
  og_title: document.querySelector('meta[property="og:title"]')?.content,
  og_desc: document.querySelector('meta[property="og:description"]')?.content,
  h1: document.querySelector('h1')?.textContent,
  url: window.location.href,
  all_text: document.body.innerText.substring(0, 8000)
})
```

### What you get

The `all_text` from `document.body.innerText` contains:
- **Chapter summaries** (章节要点) — auto-generated by Douyin, structured as timestamps + bullet points
- **Video description** and hashtags
- **Comment section** (top comments with usernames, locations, timestamps)
- **Author info** (follower count, like count)
- **Recommended videos** (sidebar)

### Key fields to extract

| Field | Source | Example |
|-------|--------|---------|
| Video title | `document.title` or h1 | "DeepSeek Agent面试分享..." |
| Author | In `all_text`, before "粉丝" | "程序员白泽【AI快速入行】" |
| Follower count | In `all_text`, "粉丝X万" | "粉丝3.6万" |
| Like count | In `all_text`, "获赞X万" | "获赞8.2万" |
| Publish date | In `all_text`, "发布时间：YYYY-MM-DD" | "发布时间：2026-06-25 22:22" |
| Chapter summaries | "章节要点" section in `all_text` | Structured bullet points with timestamps |
| Video duration | In player UI, "MM:SS / MM:SS" format | "00:53 / 05:09" |
| Comments | "全部评论" section | User comments with location and time |

### Pitfalls

- **"视频数据加载中"** — If you see this in the initial snapshot, the page hasn't loaded yet. Scroll down and wait a moment, then re-extract.
- **Login wall** — Some content may be hidden behind a login prompt, but video title, description, chapter summaries, and comments are typically visible without login.
- **Comment truncation** — `all_text` may truncate after ~8000 chars. For videos with many comments, the most important content (title + chapters + description) is at the top.
- **web_extract fails** — Douyin URLs return "Blocked: URL targets a private or internal network address" via `web_extract`. Must use the browser approach.
- **Search engine fallback** — If browser extraction fails, search for the video title on Google/DuckDuckGo to find reposts or discussions that contain the content.

## Proven workflow

```
1. browser_navigate(url)           → redirects to douyin.com/video/<id>
2. browser_scroll(direction=down)  → triggers lazy load
3. browser_console(expression=...) → extract document.body.innerText
4. Parse chapters, comments, metadata from the text
5. web_search() for additional context on the topic
6. Synthesize into research report
```

## X/Twitter post extraction (same technique)

The browser + console technique also works for X/Twitter posts. When a user shares `https://x.com/i/status/<id>` or `https://x.com/<user>/status/<id>`:

1. `browser_navigate(url)` — X loads the post page
2. `browser_console(expression=...)` — extract post text:

```javascript
JSON.stringify({
  title: document.title,
  all_text: document.body.innerText.substring(0, 8000)
})
```

The `all_text` contains the full post text, author info, engagement counts (views, replies, reposts, likes, bookmarks), and timestamps. Links in the post appear as full URLs in the text — useful for following up on referenced GitHub repos or resources.

## Research report workflow from video/social media content

When the user asks to "研究一下" or "做个报告" based on a Douyin/X link, the proven workflow is:

1. Extract content (browser + console, as above)
2. `web_search()` for additional context, background facts, and multiple sources
3. Synthesize into a structured research report with:
   - Event overview with factual data
   - Content summary (video/author points)
   - Objective analysis (what's correct, what's debatable)
   - Background context from web research
   - Investment/strategic implications
4. Write report as Markdown to `/tmp/report.md`
5. Convert to DOCX via `scripts/md_to_docx.py` (or PDF via Chrome headless)
6. Deliver via `MEDIA:/path/to/output.docx`

This workflow was proven across 4+ research reports in a single session, covering financial analysis, tech industry analysis, and career/interview guidance.

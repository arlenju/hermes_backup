## Gateway SIGTERM Recovery (2026-06-08)

**Symptom**: User reported WeChat showing "暂时无法连接". Gateway had been killed by SIGTERM at 19:29.

**Diagnosis**:
1. `cat ~/.hermes/gateway_state.json` → `gateway_state: "draining"`
2. `ps aux | grep gateway` → no process running
3. `tail -30 ~/.hermes/logs/gateway.log` → `Received SIGTERM — initiating shutdown`, `under_systemd=yes`

**Fix**:
```bash
source ~/.hermes/hermes-agent/venv/bin/activate
hermes gateway run --replace
```

## Feishu ID Confusion

**Issue**: User provided `cli_aaaf932c33b8dbea` as their user ID, but `cli_xxx` is an app_id format.

**Resolution**: The user's actual open_id was found by checking gateway logs for incoming messages:
```
grep "Inbound dm message received" gateway.log | head -1
```
→ `sender=user:ou_832ad8db9bd974c75649e86593776999`

**Feishu ID types for reference**:
- `ou_xxx` — open_id (app-scoped user ID)
- `u_xxx` — user_id (tenant-scoped)
- `on_xxx` — union_id (developer-scoped, cross-app)
- `cli_xxx` — app_id (application credential, not a user)
- `oc_xxx` — chat_id (DM or group chat)

**Send message target**: `feishu:oc_xxxxx` works for DM chats.
